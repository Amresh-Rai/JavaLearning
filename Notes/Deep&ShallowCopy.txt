तुम DSA और Java OOP समझ रहे हो, इसलिए **Shallow Copy और Deep Copy** को थोड़ा **deeply और clear mental picture** के साथ समझते हैं। यह concept **objects + references + memory** से जुड़ा है।

---

# 1. Copy का मतलब क्या है?

Copy का मतलब है **एक object से दूसरा object बनाना**।

Example:

```java
Student s1 = new Student();
Student s2 = ?   // s1 की copy
```

अब question है:

**s2 क्या s1 का independent copy होगा या दोनों same data share करेंगे?**

यहीं से दो concepts आते हैं:

* **Shallow Copy**
* **Deep Copy**

---

# 2. Shallow Copy क्या होता है

**Definition (Interview ready):**

> Shallow copy creates a new object but copies references of nested objects instead of creating new ones.

मतलब:

* outer object नया बनता है
* लेकिन **inner objects same रहते हैं**

---

## Example Structure

```java
class Address {
    String city;
}

class Student {
    String name;
    Address address;
}
```

यहाँ:

```
Student
 ├── name
 └── address (object)
```

---

# 3. Shallow Copy Example

```java
Address addr = new Address();
addr.city = "Delhi";

Student s1 = new Student();
s1.name = "Rahul";
s1.address = addr;

Student s2 = s1;   // shallow copy
```

अब memory को visualize करो।

```
s1 ---------
            \
             ----> Student Object
            /          |
s2 --------            |
                      Address Object
                      city = Delhi
```

**Important:**

* Student object same
* Address object same

---

## अब अगर change करें

```java
s2.address.city = "Mumbai";
```

अब result:

```
s1.address.city = Mumbai
```

क्यों?

क्योंकि **दोनों same Address object को point कर रहे हैं**।

---

# 4. Shallow Copy Memory Visualization

```
Stack
-----
s1
s2

Heap
-----
Student Object
  name = Rahul
  address -----> Address Object
                     city = Delhi
```

दोनों references same object को use करते हैं।

---

# 5. Deep Copy क्या होता है

**Definition (Interview ready):**

> Deep copy creates a completely independent copy of the object including all nested objects.

मतलब:

* outer object नया
* inner objects भी नए

---

# 6. Deep Copy Example

```java
Student s2 = new Student();
s2.name = s1.name;

Address newAddr = new Address();
newAddr.city = s1.address.city;

s2.address = newAddr;
```

अब memory:

```
s1 ------> Student ----> Address (Delhi)

s2 ------> Student ----> Address (Delhi)
```

दोनों objects **completely independent हैं**।

---

## अब change करें

```java
s2.address.city = "Mumbai";
```

Result:

```
s1.address.city = Delhi
s2.address.city = Mumbai
```

अब data अलग है।

---

# 7. Deep Copy Memory Diagram

```
Stack
-----
s1
s2

Heap
-----
Student1
  address ----> Address1 (Delhi)

Student2
  address ----> Address2 (Delhi)
```

अब **दोनों address objects अलग हैं**।

---

# 8. Simple Real Life Example

## Shallow Copy

सोचो:

दो students **same notebook share कर रहे हैं**।

अगर एक student कुछ change करे:

```
दोनों की notebook change हो जाएगी
```

---

## Deep Copy

अब सोचो:

दोनों students की **अपनी-अपनी notebook है**।

```
एक change करेगा → दूसरे पर effect नहीं
```

---

# 9. Shallow vs Deep Copy Table

| Feature          | Shallow Copy   | Deep Copy       |
| ---------------- | -------------- | --------------- |
| Outer Object     | New            | New             |
| Inner Objects    | Same reference | New objects     |
| Memory           | Shared         | Independent     |
| Effect of change | Affect both    | Affect only one |

---

# 10. Important Interview Point

Java में:

```
Student s2 = s1;
```

यह **copy नहीं है**।

यह सिर्फ **reference copy है**।

इसलिए technically:

```
Shallow copy भी नहीं
```

यह सिर्फ:

```
aliasing
```

है।

---

# 11. Java में Shallow Copy कैसे बनाते हैं

Normally:

* `clone()`
* copy constructor

Example:

```java
Student s2 = new Student(s1);
```

---

# 12. Super Short Summary (Remember this)

```
Shallow Copy
→ new object
→ references copied

Deep Copy
→ new object
→ inner objects also copied
```
